Initial Cleaning & Type ConversionColumn Standardization: Strips whitespace from column names.Currency Cleaning: Removes $ symbols from Quantity, Unit_Price, and Total.Numeric Conversion: Coerces strings to numbers; non-numeric values are filled with 0.Row Filtering: Drops rows where Quantity, Unit_Price, and Total are all missing or zero (invalid transactions).2. Logical Imputation (Mathematical Recalculation)The script uses the relationship $$Total = Quantity \times Unit\_Price$$ to fill in missing data:‚úÖ Missing Quantity: Calculated as $Total / Unit\_Price$.‚úÖ Missing Unit Price: Calculated as $Total / Quantity$.‚úÖ Total Recalculation: Ensures consistency by recalculating the Total for all valid rows.3. Item Mapping & CorrectionFixes invalid entries in the Item column (e.g., "ERROR", "UNKNOWN", or typos) by inferring the product based on the standardized Unit_Price:Price 1.0 $\rightarrow$ CookiePrice 1.5 $\rightarrow$ TeaPrice 2.0 $\rightarrow$ CoffeePrice 5.0 $\rightarrow$ SaladNote: Prices of 3.0 and 4.0 are explicitly marked as Unknown if the item name is invalid.4. Text Data Cleaning (Location & Payment)Targets Location and Payment_Method columns.Standardizes text using Title Case (e.g., "buDapest" $\rightarrow$ "Budapest").Uses Regex to identify variations of null values ("error", "nan", "unknown") and standardizes them to a single "UNKNOWN" label.5. Date HandlingConverts the Date column to datetime objects (handling dayfirst format).Coerces errors and fills missing dates with a default fallback value (2000-12-31) to maintain data integrity.6. Final ValidationDeduplication: Removes exact duplicate rows.Transaction ID: Drops any rows where the Transaction ID is missing, as these are not valid records.üìä OutputThe output is a CSV file (transformed_data.csv) containing structured data with consistent formatting, no critical missing values, and corrected product categorizations.
